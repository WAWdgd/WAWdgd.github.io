如何打造一款爆款博客_01.准备及初始化hexo

发表于 2019-06-29 | 分类于 前端
字数统计: 1.1k | 阅读时长 ≈ 4

制作这个博客，你的成本为0(自带二级域名,不包括自定义域名成本)，如果你第一次接触hexo，你应该会花去3小时，其中半小时用以初始化hexo，剩下的时间是各种调整工作，之后你就可以在你的网站发布你的第一篇文章。首先让我们来认识下需要的工具：

什么是hexo？

hexo是一个快速、简洁且高效的博客框架。Hexo使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用自其它地方下载的靓丽的主题生成静态网页，并将你的博客送上因特网。（来自官网）

你可以用hexo来搭建你的博客，这是目前最为主流的博客框架，以其灵活性高，安装方便而广受青睐，接下来我将用hexo教你搭建你自己的博客，这会非常简单，你只需要按部就班即可，但同时因为其强大的扩展性，你也可以在搭建完成后根据自己的喜好进行二次制作。

制作这个博客我们需要准备？

由于Hexo是一款基于Node.js的静态博客框架，hexo的运行需要你的电脑中安装有Node.js(点击即可下载)，同时注意在安装时Node.js会包含环境变量及npm的安装。安装后，检测Node.js是否安装成功，在命令行(win键+R键，输入cmd后回车)中输入 node -v，如图：

如果你看见版本号即安装成功，如果没看见，就重新安装一次；

现在环境已经配置好了，接下来我们需要安装git。什么是Git ?简单来说Git是开源的分布式版本控制系统，用于敏捷高效地处理项目，如果不懂，这些你暂时不需要理解。

首先你需要从官网下载git(点击即可下载),下载安装后在命令行中输入git查看是否成功：

这样即为安装成功。

之后将你的git于github绑定起来，如果没有github账号，去到github官网注册，这里不做过多阐述。有了github账号后，在任意文件夹右键，选择“Git Bash”

打开后输入：

git config --global user.name "你的GitHub用户名"

再输入：

git config --global user.email "你的GitHub注册邮箱"

然后直接三个回车即可。

然后进入C:\Users(用户)(系统用户名).ssh，用记事本打开id_rsa.pub，复制里面的所有内容。

之后打开github的密钥设置页面，选择右上角的New SSH key。

Title随意填写即可，将刚才复制的内容粘贴到Key中，最后点击Add SSH key。

我们再回到Git Bash中，输入 ssh git@github.com：

如上图及代表设置成功。

这里之所以设置GitHub密钥原因是，通过非对称加密的公钥与私钥来完成加密，公钥放置在GitHub上，私钥放置在自己的电脑里。GitHub要求每次推送代码都是合法用户，所以每次推送都需要输入账号密码验证推送用户是否是合法用户，为了省去每次输入密码的步骤，采用了ssh，当你推送的时候，git就会匹配你的私钥跟GitHub上面的公钥是否是配对的，若是匹配就认为你是合法用户，则允许推送。这样可以保证每次的推送都是正确合法的。

自此，准备工作结束。

打开后使用npm命令安装Hexo，输入：

安装时间可能比较长，请耐心等待。安装好后，我们需要初始化博客，博客的配置文件才会显示，输入：
